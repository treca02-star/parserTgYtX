# ParserTgYtX

Приватный Telegram-бот для сбора, AI-фильтрации и передачи криптоконтента в
редакторскую группу. MVP принимает посты из Telegram-канала и новые видео заданных
YouTube-каналов. Интеграция X/Twitter зарезервирована, но намеренно не включена.

## Возможности

- Telegram webhook и прием постов из одного входящего канала.
- YouTube WebSub: новые видео приходят без постоянного polling.
- AI-оценка через OpenRouter с четырьмя уровнями строгости и своим промптом.
- Приватная очередь владельца с кнопками `Сделать пост` и `Ссылка`.
- Кнопка `Скачать` для Telegram-видео и YouTube-роликов.
- Идемпотентная передача исходника или URL в группу обработки.
- Управление YouTube-источниками и фильтром через кнопочное меню.
- PostgreSQL, Alembic, Docker Compose, health endpoint.

## Подготовка Telegram

1. Создайте бота через `@BotFather`.
2. Добавьте бота администратором во входящий канал и группу обработки.
3. Во входящем канале разрешите получать публикации.
4. В группе обработки разрешите публиковать сообщения.
5. Узнайте собственный Telegram user ID и ID обоих каналов.

Бот отвечает только пользователю из `TELEGRAM_OWNER_ID`. Для публичного
Telegram-канала карточка содержит прямую ссылку. Для приватного канала ссылка
может отсутствовать, но доступное сообщение все равно копируется в группу обработки.

## Запуск на VPS

Требуются Docker, Docker Compose, домен и HTTPS reverse proxy.

```bash
cp .env.example .env
# заполните .env реальными значениями
docker compose up -d --build
curl https://parser.example.com/health
```

`PUBLIC_BASE_URL` должен быть публичным HTTPS-адресом: Telegram и YouTube
должны обращаться к webhook извне. Проксируйте этот адрес на порт `8000`.

Пароль PostgreSQL задается одновременно:

```env
POSTGRES_PASSWORD=strong-password
DATABASE_URL=postgresql+asyncpg://parser:strong-password@db:5432/parser
```

## Работа

1. Откройте личный чат с ботом и отправьте `/start`.
2. Через `📺 Источники` добавьте URL YouTube-каналов.
3. Через `🎯 AI-фильтр` выберите строгость и задайте критерии.
4. Пересылайте материалы вашим готовым сборщиком во входящий канал.
5. В карточке нажмите `✅ Сделать пост`, чтобы отправить исходник в обработку.

YouTube WebSub-подписки продлеваются при запуске приложения. Рекомендуется
перезапускать сервис не реже одного раза в неделю либо добавить внешний
плановый restart.

## Локальная разработка

```bash
python -m venv .venv
.venv/Scripts/activate
pip install -e ".[dev]"
pytest
ruff check .
mypy app
```

Для локального запуска нужен PostgreSQL и публичный HTTPS-туннель.

## Безопасность

- Никогда не коммитьте `.env`.
- Используйте длинный случайный `TELEGRAM_WEBHOOK_SECRET`.
- Ограничьте сетевой доступ к PostgreSQL.
- Делайте ежедневный `pg_dump` и храните копии вне VPS.
- Не публикуйте вывод логов с токенами.

## Ограничения MVP

- Один владелец, один входящий Telegram-канал и одна группа обработки.
- Защищенные Telegram-сообщения могут не копироваться ограничениями Telegram.
- При ошибке OpenRouter материал показывается владельцу с резервным заголовком.
- X/Twitter будет добавлен отдельным официальным API-адаптером.
- YouTube-файлы отправляются в качестве, которое помещается в лимит 49 МБ.
